# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository purpose

Content repository for the author's [Zenn](https://zenn.dev) tech blog, managed via [zenn-cli](https://zenn.dev/zenn/articles/zenn-cli-guide). There is no application code — each file under `articles/` is a standalone Markdown post that Zenn renders from GitHub.

## Commands

```bash
# Create a new article (generates articles/<random-slug>.md with frontmatter)
npx zenn new:article

# Create a new book
npx zenn new:book

# Local preview server (renders articles as Zenn would)
npx zenn preview
```

There are no build, lint, or test steps — `package.json`'s `test` script is a placeholder.

## Article layout

Each article is `articles/<14-char-slug>.md`. The slug is the Zenn article ID and must not be renamed once published (it is the canonical URL path). Required frontmatter:

```md
---
title: ""           # article title
emoji: "😸"          # single emoji used as thumbnail
type: "tech"        # "tech" (technical) or "idea" (opinion/idea)
topics: []          # tag list, e.g. ["go", "aws"]
published: true     # false = draft, not visible on zenn.dev
---
```

Optional: `published_at: "YYYY-MM-DD HH:MM"` to backdate, `publication_name` to publish under a Zenn Publication.

Publishing happens automatically when commits land on `main` and Zenn syncs from GitHub — there is no deploy step in this repo.

## Conventions observed in existing articles

- Japanese body text; code blocks and technical terms in English.
- Slugs are generated by `zenn new:article` — do not hand-author them.
- Images go alongside articles (e.g. `articles/image.png`) and are referenced with relative paths.
