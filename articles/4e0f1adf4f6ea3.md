# 目的

- エンジニアとして生産性を上げるために意識していることを残しておきたい&都度更新していきたい
- ここら辺意識して仕事しておけば、ある程度仕事できるやつになれるんじゃないか説を立証したい

# 意識していることリスト (※全部できているわけではない)

## ✅ 優先順位を決めて、きっちり守る

- 勤務中は色んな人から連絡がきたりするので、作業効率を落とさないように優先順位を決めている
- プロダクトを前に進めること = リリースすることを基準に決めている
    - 会社としての優先順位、チームとしての優先順位と同じようになるようにしている
- 自分の中で決めている優先順位
    1. ユーザー影響のある機能の不具合/バグ
    2. PMやエンジニア、デザイナーからのメンション (相談とか仕様の確認とかのメッセージ)
    3. 他の人のPRのレビュー
    4. 自分のPRのレビュー指摘対応
    5. 自分のタスクの実装
- 補足
    - 1に関しては判断がつかない場合もあるのでPM等に相談することもある
    - 2, 3に関しては**他の人のタスクを止めないということで優先順位を高くしている**
- ただ決めるだけではなく、きっちり実行に移すことが大切

## ✅ 発言する

- リモートで顔が見えない場合、話を聞いていることを相手にちゃんと示す
    - スタンプでもいいので反応をちゃんと返す
- グループメンションされた時も一旦反応してみる = 放置しないことを意識する
    - 「みます」とか「考えます」という1次返答だけでも良いと思う
    - 反応しないで無視してるみたいになったり、リマインダーみたいな形で再度連絡が来ないようにしている
    - 絵文字スタンプだけ押しても人によって捉え方が違うので**必ず文字として反応する**
- 自分が尊敬するエンジニア、できるエンジニアはみんなのいる場でちゃんと発言している (できる) イメージがある
- **発言する練習 = 自分の考えを言語化する練習 = 自分の意図ややりたいことを周囲に上手に伝える**ことができるようになる

## ✅ 周りの人と目線を合わせる

- 手戻りしないように、常に周りの人と目線を合わせる/共通認識を持つ
- 周りの人 = PM/デザイナー/エンジニア
- そのために
    - おや？と思うことがあれば、エンジニア/PM/デザイナーに聞く
    - ドキュメントにコメントする
    - ドキュメントを更新する
    - **言葉の定義を明確にする**
        - バックグラウンドが違うので同じ言葉でも違う場所/概念を指していることもあるので
        - 議論が平行線にならないように共通認識を持つ
- チームで仕事しているんだから、チームでアウトプットを出せるように知識とか情報は基本共有した方が良いよねって話

## ✅ イメージする

- 原稿を書くときに下書きするように、実際にコードを書く前に実装をイメージする
    - コードはどこに置くか？
    - コードはどこまで分割するか？
    - コンポーネントはどうやって構成するか？
    - こういうエラーだったらフロントの挙動はどうなるか？
    - こういう動作をしたい場合はAPIはどういう情報を返せばよいか？
    - etc…
- 手を動かす前にイメージすることができれば、それだけでも手戻りを少なくできる
- 頭の中で完結させるのが難しければ、図に書いてみる

## ✅ 分からないことを分からないままにしない

- どうやってバグを見つけたのか？この概念は何か？この機能は何か？など
    - **次に自分が同じ状況になったら同じようにできるかを考えながら人の作業を見る/レビューする**
- 分からないままだと何もできないので、人に聞いて理解する
    - ツールの使い方やログの見方とか
- 人に聞けない場合は調べる、よく言われるようにググり力大切
    - このことは自分が初めて知った、ということはほぼないので先人たちの残したサイトを頑張って見つけ出す
- どうしても分からなければ、サイトで質問してみる
    - stack overflowに書くと秒速で解答が返ってくるのでおすすめ

## ✅ モチベーションで仕事をしない

- モチベーションによって業務スピードが落ちたり、質が落ちる社会人は2流 (とどこかのCOOが言ってた)
- 言い換えると、1流のエンジニアはモチベーションに左右されず、いかなる時も質の高いプロダクトを素早く開発できる (でも現実的には難しいよね)

## ✅ 開発環境を心地いいものにする

- 無駄なアプリを使わない
    - 音楽のアプリとか業務に関係ないアプリは常時開かないとか
    - アプリを使ったらその都度終了させるとか
- スニペットを登録したり、ショートカットを登録する
    - ミスしやすいものとかはテンプレート化してしまう
- Vimのキーバインディングを学ぶ
    - emacsでもいい
    - 矢印キーやdeleteキーを使わずにコーディングする
    - ファイルの移動やコピペなどをキーだけで完結させる
- ヘッドホン等をして外部からの雑音を排除する

## ✅ 機能をさわる

- 自分が開発した機能以外の機能を触ってみる
- 自分が知らない機能があったら、どんなものか軽く調べてみる
    - 次やるタスクが似たようなものであれば、簡単に実装できる
    - 単純に手数が増える
- 普段からアプリを触る癖があれば、異常に気付ける
